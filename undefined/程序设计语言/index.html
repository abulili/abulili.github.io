程序设计语言

编译程序和解释程序

低级语言

  • 硬件只能识别由0、1组成的机器指令序列（程序）

    • 机器指令是最基本的计算机语言

  • 用符号表示的指令为汇编指令，汇编指令的集合为汇编语言，ADD表示加

    • 书写格式取决于特定计算机的机器指令

高级语言

  • 变为低级语言：汇编/解释/编译 进行翻译

    • 编译程序：源程序翻译成目标语言程序，在计算机上运行目标程序

      • 不参与，翻译成独立保存的目标程序
      • java

    • 解释程序（解释器）：直接解释执行源程序/源程序翻译成中间代码后再执行

      • 源程序参与运行，所以慢

  • 解释脚本语言：PHP/Python/JS

高级语言变低级的方式

  • 编译方式

    • 词法分析、语法分析、语义分析、（中间代码生成、代码优化）[可省略]、目标代码生成

      • 只涉及到符号表管理+出错管理

        • 不断收集、记录和使用源程序中一些相关符号的类型的特征信息，并将其存入符号表中。
        • 记录源程序中各个字符的必要信息，以辅助语义的正确性检查和代码生成

  • 解释方式

    • 上述前三个，这三个都不可省略+顺序不可交换

编译过程（源程序（看作多行字符串）先进入）

  • 词法分析

    • 根据语言的词法规则，分析构成程序的字符，及由字符按照构造规则构成的符号是否符合程序语言的规定

    • 扫描出”单词符号“

    • 记号流（输出）

      • 二元组方式：单词种别和单词自身的值

  • 语法分析

    • 记号流（输入）

    • 工具

      • 有限自动机，能正确识别正规集

        • 分类

          • 确定的有限自动机（DFA）

            • 对每一个状态来说识别字符后转移的状态是唯一的

              • 0、1状态唯一

          • 不确定的有限自动机（NFA）

            • 对每一个状态来说识别字符后转移的状态是不唯一的

        • 3 可以不传东西

          • 到终态还能走，就是要走到最后都是终态才行

            • 先看必须带谁

      • 上下文无关文法

        • 用于表示各种程序设计语言的语法规则

        • 小写a/b/c表示终结符号

        • 核心部分是产生式集合，文法简写为其产生式集合

          • E是开始符号，N是终结符号，T是产生式集合，F是非终结符号

            -

            -

          • 终结符号集：不能继续往下推

            • 产生式：里面含有非终结符号

              • 推的终结态还要按顺序，里面的符号不能想当然改变，一直是位置符号都不变，终结态如果是字母表示的变量，那就不能出现数字

    • 作用

      • 对各条语句的结构进行合法性分析

      • 分析程序中的句子结构是否正确

        • 逗号、分号之类对不对

      • 发现程序中所有语法错误

    • 构造语法树（分析树）（输出）

      • 有错误，语法错误指出处
      • 例如括号匹配

  • 语义分析

    • 构造语法树（分析树）（输入）

    • 类型分析和检查

      • 分析各语法结构的含义，检查源程序是否包含静态语义错误（运行之前）

        • 动态语义错误（抛异常）：死循环、零除数、数字下标越界、堆栈溢出、指针异常

      • 类型的载体及其上的运算

        • 取余只能对整型数据进行运算，否则认为类型不匹配错误

    • 对各条语句的结构进行合法性分析，分析程序中的句子结构是否正确

    • 不能发现程序中所有语义错误，只能静态错误，动态语义错误只有运行时才能发现

  • （中间代码生成）

  • （代码优化）

  • 目标代码生成

    • 把中间代码变换成特定机器上的绝对指令代码、可重定位的指令代码或汇编指令代码
    • 这个阶段的工作与具体的机器密切相关
    • 寄存器的分配处于目标代码生成阶段

程序基本

三种控制结构：顺序、选择、循环（重复）

隐式转换，小的数据范围向大的数据范围看齐，int->double，但是赋值按被赋值类型算int=(int)double

常量没有对应存储单元，常量区

程序设计语言的基本成分包括数据、运算、控制、传输

函数调用

传值调用：实参（变量/常量/表达式）的值传给形参，形参应该是一个变量名

传值调用：实参（仅变量）的地址给形参，实参、形参必须有地址

后缀式(ab?)、中缀式(a?b）

优先级 （）x / +、-，优先级相同从右向左

中序遍历-中缀式（左根右），后序遍历-后缀式（左右根）
